# บทที่ 2 ทบทวนวรรณกรรมและงานวิจัยที่เกี่ยวข้อง

บทนี้ทบทวนวรรณกรรมและงานวิจัยที่เกี่ยวข้องกับการพัฒนาระบบ VIRIYA เพื่อวางรากฐานเชิงทฤษฎีและเทคนิคสำหรับการออกแบบและการประเมินระบบ เนื้อหาครอบคลุม 7 ด้าน ได้แก่ โมเดลภาษาขนาดใหญ่และ Generative AI (2.1) เทคนิค Retrieval-Augmented Generation (2.2) การสืบค้นแบบเวกเตอร์และการจัดลำดับใหม่ (2.3) สถาปัตยกรรมแบบ Multi-tenant Software-as-a-Service (2.4) แบบจำลองการยอมรับเทคโนโลยี (2.5) การนำแชทบอทไปใช้ในบริบทการศึกษา (2.6) และสรุปเป็นกรอบแนวคิดของการศึกษา (2.7)

## 2.1 โมเดลภาษาขนาดใหญ่และ Generative AI

### 2.1.1 สถาปัตยกรรม Transformer

รากฐานของโมเดลภาษาขนาดใหญ่ในปัจจุบันมาจากสถาปัตยกรรม Transformer ที่ {Vaswani, 2017 #6} นำเสนอในงานประชุม NeurIPS แนวคิดหลักคือการแทนที่การประมวลผลลำดับของข้อความแบบ recurrent และ convolutional ด้วยกลไก self-attention ทั้งหมด ซึ่งทำให้โมเดลสามารถพิจารณาความสัมพันธ์ระหว่างโทเค็นทุกตัวในลำดับได้พร้อมกัน การออกแบบนี้ทำให้การฝึกโมเดลขนาดใหญ่บนข้อมูลข้อความปริมาณมหาศาลเป็นไปได้ในเชิงปฏิบัติ และเป็นรากฐานของโมเดลในตระกูล BERT, GPT และ Claude ทั้งหมด

ผลการทดสอบในงานของ {Vaswani, 2017 #6} แสดงว่า Transformer สามารถทำคะแนน BLEU ที่ 28.4 ในการแปลภาษาอังกฤษเป็นเยอรมัน และ 41.8 ในภาษาอังกฤษเป็นฝรั่งเศส ซึ่งเหนือกว่าผลงานที่ดีที่สุดในขณะนั้น พร้อมทั้งใช้เวลาฝึกสั้นกว่ามาก ผลกระทบของงานวิจัยนี้สะท้อนผ่านจำนวนการอ้างอิงที่เกินหนึ่งแสนครั้งในปี 2568 ทำให้ Transformer เป็นสถาปัตยกรรมมาตรฐานของงานประมวลผลภาษาธรรมชาติในยุคหลังปี 2560

### 2.1.2 ตระกูลโมเดล Claude ของ Anthropic

โมเดลที่ใช้ในระบบ VIRIYA เป็นตระกูล Claude ของบริษัท Anthropic ซึ่งเปิดตัวครั้งแรกในปี 2567 ในรุ่น Claude 3 ที่ประกอบด้วยสามขนาด คือ Opus, Sonnet และ Haiku โดย {Anthropic, 2024 #10} ระบุใน model card ว่าโมเดลทั้งสามได้รับการออกแบบให้รองรับงานที่มีระดับความซับซ้อนแตกต่างกัน Opus สำหรับงานที่ต้องการการใช้เหตุผลในระดับลึก Sonnet สำหรับงานทั่วไปที่ต้องการสมดุลระหว่างความเร็วและคุณภาพ และ Haiku สำหรับงานที่ต้องการความเร็วสูงและต้นทุนต่ำ

ในรุ่นต่อมาที่เปิดตัวในปี 2568 และ 2569 ตระกูล Claude เพิ่มความสามารถด้านการใช้เหตุผลแบบขยาย (Extended Thinking) ที่ทำให้โมเดลสามารถวางแผนการตอบในหลายขั้นตอนก่อนการสร้างคำตอบสุดท้าย ความสามารถนี้มีประโยชน์กับงานที่ต้องการการวิเคราะห์เอกสารขนาดใหญ่ เช่น การแปลงเอกสาร PDF ที่มีโครงสร้างซับซ้อนให้เป็น chunk ตามที่ระบบ VIRIYA ใช้งานในกระบวนการ ingestion ของเอกสาร

### 2.1.3 ข้อจำกัดของโมเดลภาษาขนาดใหญ่

แม้โมเดลภาษาขนาดใหญ่จะมีความสามารถในการตอบคำถามในหลายรูปแบบ แต่ยังมีข้อจำกัดสำคัญสองประการที่ส่งผลต่อการนำไปใช้งานในบริบทที่ต้องการความแม่นยำ ข้อจำกัดแรกคือการสร้างข้อมูลเท็จ (Hallucination) ที่โมเดลอาจตอบคำถามด้วยเนื้อหาที่ฟังดูเป็นธรรมชาติแต่ไม่ตรงกับความจริง โดยเฉพาะเมื่อถูกถามเรื่องที่ไม่อยู่ในข้อมูลฝึก ข้อจำกัดที่สองคือข้อมูลของโมเดลมีจุดตัด (Knowledge Cutoff) ทำให้โมเดลไม่รู้เหตุการณ์หรือเอกสารที่เกิดขึ้นหลังจากวันที่ฝึก ข้อจำกัดทั้งสองประการนี้เป็นสาเหตุหลักที่ทำให้เทคนิค Retrieval-Augmented Generation ได้รับความสนใจในการแก้ปัญหา ตามที่อธิบายในหัวข้อถัดไป

## 2.2 Retrieval-Augmented Generation (RAG)

### 2.2.1 แนวคิดพื้นฐาน

{Lewis, 2020 #1} นำเสนอเทคนิค Retrieval-Augmented Generation (RAG) เพื่อแก้ปัญหาข้อจำกัดของโมเดลภาษาขนาดใหญ่ โดยผสานส่วนการสืบค้นเอกสารจากฐานความรู้ภายนอก (non-parametric memory) เข้ากับโมเดลภาษาขนาดใหญ่ (parametric memory) ในกระบวนการเดียวกัน เมื่อผู้ใช้งานถามคำถาม ระบบจะแปลงคำถามเป็นเวกเตอร์แล้วสืบค้นเอกสารที่เกี่ยวข้องจากฐานข้อมูลก่อน จากนั้นจึงส่งทั้งคำถามและเอกสารที่สืบค้นได้ให้โมเดลภาษาขนาดใหญ่สร้างคำตอบโดยอ้างอิงจากเอกสารเหล่านั้น

แนวคิด RAG มีข้อดีที่สำคัญหลายประการ ประการแรกคือการลดปัญหา hallucination เนื่องจากโมเดลสร้างคำตอบจากบริบทของเอกสารที่มีอยู่จริงแทนการใช้ความรู้ภายในของโมเดลเพียงอย่างเดียว ประการที่สองคือการอัปเดตความรู้ทำได้โดยการเพิ่มเอกสารใหม่เข้าสู่ฐานข้อมูลโดยไม่ต้องฝึกโมเดลใหม่ ประการที่สามคือความสามารถในการอ้างอิงแหล่งที่มา (citation) ทำให้ผู้ใช้งานสามารถตรวจสอบความถูกต้องย้อนกลับได้

### 2.2.2 การวิวัฒนาการจาก Naive RAG สู่ Advanced RAG

{Gao, 2024 #8} จัดกลุ่มกระบวนทัศน์ของ RAG ออกเป็นสามรุ่นตามระดับความซับซ้อน Naive RAG เป็นรุ่นพื้นฐานที่ใช้ขั้นตอนสามส่วน ได้แก่ การแบ่งเอกสาร (chunking) การแปลงเป็นเวกเตอร์ (embedding) และการสืบค้นโดยใช้ความคล้ายคลึงของเวกเตอร์ (similarity search) เพียงรอบเดียวก่อนสร้างคำตอบ Advanced RAG เพิ่มเทคนิคการเพิ่มประสิทธิภาพก่อนการสืบค้น (pre-retrieval) และหลังการสืบค้น (post-retrieval) เช่น การเขียนคำถามใหม่ (query rewriting) การใช้หลายเวอร์ชันของคำถาม (multi-query) และการจัดลำดับใหม่ (reranking) ด้วย cross-encoder สุดท้าย Modular RAG ปรับโครงสร้างให้มีความยืดหยุ่นในการประกอบส่วนประกอบต่าง ๆ ตามบริบทและประเภทของคำถาม

ระบบ VIRIYA ใช้แนวทาง Advanced RAG ร่วมกับองค์ประกอบของ Modular RAG โดยมีการเขียนคำถามใหม่ การสร้างหลายเวอร์ชันของคำถาม การสืบค้นแบบ hybrid การจัดลำดับใหม่ด้วย cross-encoder และการใช้ agent loop ในการตัดสินใจว่าข้อมูลที่สืบค้นได้เพียงพอสำหรับการตอบหรือไม่

### 2.2.3 Agentic RAG

การขยายแนวคิด RAG ไปสู่ Agentic RAG มาจากการมอง LLM เป็น agent ที่สามารถเลือกใช้เครื่องมือ (tools) ในการสืบค้น ประเมินผลลัพธ์ และตัดสินใจเรียกใช้เครื่องมือเพิ่มเติมได้ตามความจำเป็น รูปแบบนี้แตกต่างจาก RAG แบบ single-pass ที่สืบค้นเพียงครั้งเดียวก่อนสร้างคำตอบ โดย Agentic RAG สามารถจัดการคำถามที่ซับซ้อนที่ต้องการข้อมูลหลายส่วนหรือต้องการการสืบค้นซ้ำได้ดีกว่า

{Gao, 2024 #8} ระบุว่างานวิจัยในทิศทาง Agentic RAG กำลังได้รับความสนใจอย่างกว้างขวางในปี 2567 และกำลังก่อตัวเป็นทิศทางมาตรฐานสำหรับระบบที่ต้องการความเป็นเหตุเป็นผลในหลายขั้นตอน ระบบ VIRIYA นำแนวทางนี้มาปรับใช้ในรูปแบบ ReAct loop ที่ agent สามารถเรียกใช้เครื่องมือ `hybrid_search` และ `fetch_webpage` ได้ตามความจำเป็น โดยมีขีดจำกัดรอบการเรียกที่ 12 รอบต่อคำถามเพื่อป้องกันการวนซ้ำที่ไม่สิ้นสุด

## 2.3 การสืบค้นแบบเวกเตอร์และการจัดลำดับใหม่

### 2.3.1 Dense Passage Retrieval

{Karpukhin, 2020 #7} นำเสนอแนวทาง Dense Passage Retrieval (DPR) เพื่อใช้ในระบบตอบคำถามแบบ open-domain เทคนิคหลักคือการใช้สถาปัตยกรรม dual-encoder ที่ประกอบด้วยโมเดล BERT สองตัวแยกกัน ตัวหนึ่งสำหรับการเข้ารหัสคำถามและอีกตัวหนึ่งสำหรับการเข้ารหัสข้อความ (passage) การฝึกใช้วิธี metric learning ที่เพิ่มความคล้ายคลึงแบบ dot product ระหว่างคำถามและข้อความที่เกี่ยวข้อง และลดระหว่างคำถามและข้อความที่ไม่เกี่ยวข้อง

ผลการทดลองของ {Karpukhin, 2020 #7} แสดงว่า DPR สามารถเอาชนะระบบสืบค้นแบบ BM25 ของ Lucene ที่แข็งแรงในหลายชุดข้อมูลของการตอบคำถามแบบ open-domain ได้ด้วยระยะห่าง 9-19 เปอร์เซ็นต์ในตัวชี้วัดความแม่นยำของ top-20 passage retrieval แนวคิดนี้เป็นรากฐานของการใช้ฐานข้อมูลเวกเตอร์ (vector database) ในระบบ RAG สมัยใหม่

### 2.3.2 BM25 และการสืบค้นแบบ sparse

แม้ Dense retrieval จะเป็นแนวทางที่ทันสมัย แต่การสืบค้นแบบ sparse ที่อาศัยการตรงกันของคำ (lexical matching) ยังคงมีบทบาทสำคัญในบริบทที่คำถามมีคำเฉพาะเจาะจง เช่น รหัสวิชา ชื่อเฉพาะ หรือเลขรหัสนิสิต {Robertson, 2009 #12} วิเคราะห์กรอบความน่าจะเป็นของการค้นคืนเอกสาร (Probabilistic Relevance Framework) และอธิบายอัลกอริทึม BM25 ซึ่งเป็นหนึ่งในอัลกอริทึมการให้คะแนนความเกี่ยวข้องของเอกสารที่ประสบความสำเร็จมากที่สุดในประวัติศาสตร์ของ Information Retrieval

BM25 ให้คะแนนความเกี่ยวข้องโดยพิจารณาจากความถี่ของคำในเอกสาร (term frequency) ความหายากของคำในฐานข้อมูลทั้งหมด (inverse document frequency) และการปรับความยาวของเอกสาร (document length normalization) ในระบบ RAG สมัยใหม่ การผสานการสืบค้นแบบ dense และ sparse เข้าด้วยกันในรูปแบบ Hybrid Search ได้กลายเป็นแนวปฏิบัติที่ดีสำหรับการเพิ่มความแม่นยำในการดึงเอกสารในสถานการณ์ที่หลากหลาย ระบบ VIRIYA ใช้รูปแบบ hybrid ที่รวมผลจาก Pinecone (dense) และ BM25 (sparse) โดยใช้ Reciprocal Rank Fusion ในการรวมคะแนน

### 2.3.3 การจัดลำดับใหม่ด้วย Cross-Encoder

หลังจากสืบค้นเอกสารจำนวนหนึ่ง (เช่น top-25) การจัดลำดับใหม่ด้วยโมเดลที่ละเอียดกว่าสามารถช่วยเพิ่มความแม่นยำของเอกสารที่ส่งให้ LLM ได้ {Nogueira, 2019 #11} นำเสนอการใช้ BERT ในรูปแบบ cross-encoder เพื่อจัดลำดับใหม่โดยให้โมเดลพิจารณาคำถามและเอกสารร่วมกัน (แทนที่จะเข้ารหัสแยกกันเหมือน dual-encoder) รูปแบบนี้ให้ความแม่นยำที่สูงกว่า แต่มีต้นทุนการประมวลผลที่สูงกว่าเช่นกัน

ผลการทดลองของ {Nogueira, 2019 #11} แสดงว่าการใช้ BERT reranker สามารถเพิ่มคะแนน MRR@10 ในชุดข้อมูล MS MARCO ได้ถึง 27 เปอร์เซ็นต์เมื่อเทียบกับเทคนิคก่อนหน้า ในบริบทของระบบ RAG สมัยใหม่ cross-encoder reranker มักถูกใช้เป็นชั้นที่สองหลังจากการสืบค้นเบื้องต้นเพื่อคัดกรองเอกสารให้เหลือเฉพาะชิ้นที่เกี่ยวข้องที่สุดก่อนส่งให้ LLM ระบบ VIRIYA ใช้บริการ Cohere Rerank v3.5 ซึ่งเป็น cross-encoder ที่ปรับเฉพาะให้เหมาะกับงานนี้

## 2.4 Multi-tenant Software-as-a-Service Architecture

{Bezemer, 2010 #3} อธิบายแนวคิดของสถาปัตยกรรม Multi-tenant SaaS ในฐานะกระบวนทัศน์ทางซอฟต์แวร์ที่ระบบหนึ่งให้บริการแก่ลูกค้าหลายราย (tenants) พร้อมกัน โดยแชร์โครงสร้างพื้นฐานและฐานรหัสเดียวกัน แต่แยกข้อมูลและการตั้งค่าระหว่างลูกค้าแต่ละรายอย่างเคร่งครัด ข้อดีหลักคือการใช้ทรัพยากรได้อย่างมีประสิทธิภาพและการบำรุงรักษาที่ง่ายขึ้นเมื่อระบบเติบโต ขณะที่ข้อควรระวังคือการออกแบบสถาปัตยกรรมที่ไม่ดีสามารถทำให้การดูแลระบบกลายเป็นฝันร้าย (maintenance nightmare) ได้

ในบริบทของระบบ RAG สำหรับสถาบันการศึกษา การออกแบบ Multi-tenant จำเป็นต้องพิจารณาสองระดับของการแยกข้อมูล ระดับแรกคือการแยกเอกสารและเวกเตอร์ของแต่ละหลักสูตรไม่ให้ปะปนกันเมื่อค้นหา ซึ่งสามารถทำได้ผ่านกลไก namespace ในฐานข้อมูลเวกเตอร์ ระดับที่สองคือการแยกข้อมูลการสนทนา ข้อมูลผู้ใช้งาน และการตั้งค่า tenant ในฐานข้อมูลหลัก ซึ่งในระบบ VIRIYA ใช้ `tenant_id` เป็นคีย์ประกอบในทุกเอกสารของ Firestore ตามแนวทางที่ {Bezemer, 2010 #3} ระบุไว้เป็น data isolation pattern

ผลที่เกิดขึ้นจากการออกแบบ Multi-tenant ที่ดีคือการสามารถเพิ่ม tenant ใหม่เข้าสู่ระบบได้โดยไม่ต้องเปลี่ยนแปลงโครงสร้างพื้นฐาน ในระบบ VIRIYA การเพิ่มหลักสูตรใหม่ทำได้เพียงสร้าง tenant configuration ใหม่ใน Firestore และตั้งค่า LINE Official Account เท่านั้น โดยไม่ต้องปรับปรุงรหัสของระบบ ซึ่งเป็นเงื่อนไขสำคัญต่อความเป็นไปได้เชิงพาณิชย์ที่จะอธิบายในบทที่ 5

## 2.5 แบบจำลองการยอมรับเทคโนโลยี

### 2.5.1 Technology Acceptance Model (TAM)

{Davis, 1989 #2} นำเสนอ Technology Acceptance Model (TAM) ซึ่งเป็นแบบจำลองเชิงทฤษฎีที่ได้รับการอ้างอิงอย่างกว้างขวางในการอธิบายการยอมรับเทคโนโลยีของผู้ใช้งาน แบบจำลองระบุว่าการใช้งานจริงของระบบเทคโนโลยี (Actual Use) ถูกขับเคลื่อนโดยความตั้งใจในการใช้งาน (Behavioral Intention) ซึ่งในทางกลับกันถูกกำหนดโดยตัวแปรหลักสองตัว ได้แก่ การรับรู้ประโยชน์ (Perceived Usefulness: PU) และการรับรู้ความง่ายในการใช้งาน (Perceived Ease of Use: PEOU)

งานวิจัยของ {Davis, 1989 #2} พัฒนาเครื่องมือวัดสำหรับตัวแปรหลักทั้งสองและทดสอบใน 2 ศึกษาที่ครอบคลุมผู้ใช้งาน 152 คนและโปรแกรมประยุกต์ 4 โปรแกรม ผลการศึกษาพบว่าทั้ง PU และ PEOU มีความสัมพันธ์อย่างมีนัยสำคัญกับการใช้งานจริง แต่ PU มีความสัมพันธ์สูงกว่า PEOU แสดงว่าการรับรู้ว่าเทคโนโลยีมีประโยชน์ต่องานเป็นปัจจัยที่ขับเคลื่อนการยอมรับมากกว่าการรับรู้ว่าใช้งานได้ง่าย

### 2.5.2 การขยายกรอบ TAM สู่ TAM2

{Venkatesh, 2000 #9} นำเสนอการขยายกรอบ TAM เป็น TAM2 ที่เพิ่มตัวแปรเชิงสังคมและเชิงปัญญาเข้าสู่แบบจำลอง ตัวแปรเชิงสังคมประกอบด้วยบรรทัดฐานทางสังคม (subjective norm) ความสมัครใจ (voluntariness) และภาพลักษณ์ (image) ส่วนตัวแปรเชิงปัญญาประกอบด้วยความเกี่ยวข้องกับงาน (job relevance) คุณภาพของผลลัพธ์ (output quality) และความสามารถในการแสดงผล (result demonstrability) การขยายกรอบนี้ได้รับการทดสอบในการศึกษาแบบ longitudinal จำนวน 4 องค์กร และสามารถอธิบายความแปรปรวนของ PU ได้ในช่วง 40-60 เปอร์เซ็นต์

### 2.5.3 Unified Theory of Acceptance and Use of Technology (UTAUT)

{Venkatesh, 2003 #14} นำเสนอการพัฒนาต่อยอดโดยรวบรวมปัจจัยจากแบบจำลองการยอมรับเทคโนโลยีแปดรูปแบบก่อนหน้า (รวมถึง TAM, TAM2, Theory of Reasoned Action, Theory of Planned Behavior, Motivational Model, Model of PC Utilization, Innovation Diffusion Theory และ Social Cognitive Theory) เข้าเป็นทฤษฎีเดียวในชื่อ Unified Theory of Acceptance and Use of Technology (UTAUT) โดยระบุตัวแปรหลักสี่ประการที่ส่งผลต่อความตั้งใจในการใช้งาน (Behavioral Intention) และการใช้งานจริง (Use Behavior)

- **Performance Expectancy (PE)** การรับรู้ของผู้ใช้งานว่าการใช้ระบบช่วยเพิ่มประสิทธิภาพในการทำงาน
- **Effort Expectancy (EE)** การรับรู้ว่าการใช้ระบบทำได้ง่ายโดยไม่ต้องใช้ความพยายามมาก
- **Social Influence (SI)** การรับรู้อิทธิพลจากผู้อื่นในสภาพแวดล้อมทางสังคมของผู้ใช้งาน
- **Facilitating Conditions (FC)** การรับรู้ว่ามีทรัพยากร โครงสร้างพื้นฐาน และการสนับสนุนที่เพียงพอสำหรับการใช้งาน

นอกจากนั้น UTAUT ยังกำหนดตัวแปรปรับ (moderators) สี่ประการที่มีผลต่อความแรงของความสัมพันธ์ระหว่างตัวแปรหลักและตัวแปรตาม ได้แก่ เพศ อายุ ประสบการณ์ และความสมัครใจในการใช้งาน ผลการทดสอบ UTAUT กับข้อมูลจากสี่องค์กรพบว่าแบบจำลองสามารถอธิบายความแปรปรวนของความตั้งใจในการใช้งานได้ถึง 70 เปอร์เซ็นต์ (adjusted R²) ซึ่งสูงกว่าแบบจำลองแปดรูปแบบก่อนหน้าที่อธิบายได้ในช่วง 17-53 เปอร์เซ็นต์

ในการศึกษาของระบบ VIRIYA ผู้วิจัยเลือกใช้กรอบ UTAUT ของ {Venkatesh, 2003 #14} เป็นกรอบหลักในการประเมินการยอมรับระบบของนิสิตผู้ใช้งาน เนื่องจากกรอบนี้ครอบคลุมมิติด้านสังคม (SI) และเงื่อนไขสนับสนุน (FC) ที่กรอบ TAM พื้นฐานไม่ครอบคลุม ซึ่งเป็นประเด็นสำคัญในบริบทของสถาบันการศึกษาที่การยอมรับเทคโนโลยีได้รับอิทธิพลจากเพื่อนและรุ่นพี่รุ่นน้อง รวมทั้งความพร้อมของระบบสนับสนุนเมื่อบอทไม่สามารถตอบคำถามได้ เนื่องจากขนาดของกลุ่มตัวอย่างในการศึกษา (N=6) มีจำกัด ผู้วิจัยเลือกใช้รูปแบบการสัมภาษณ์เชิงลึกแทนการวัดด้วยแบบสอบถามเชิงปริมาณ และใช้ทั้งห้าโครงสร้างของ UTAUT เป็นกรอบในการวิเคราะห์ข้อมูลเชิงคุณภาพ (deductive thematic coding) ตามแนวทางของ {Braun, 2006 #5}

## 2.6 การนำแชทบอทไปใช้ในบริบทการศึกษา

{Labadze, 2023 #13} ทบทวนวรรณกรรมเกี่ยวกับการนำแชทบอทที่ขับเคลื่อนด้วย AI ไปใช้ในสถาบันการศึกษาอย่างเป็นระบบ โดยศึกษาเอกสาร 67 ชิ้นที่ผ่านเกณฑ์การคัดเลือก ผลการทบทวนระบุประโยชน์หลักสามด้านที่นิสิตได้รับจากระบบแชทบอท ได้แก่ การช่วยเหลือในการเรียนและงานมอบหมาย การเรียนรู้ที่ปรับเฉพาะบุคคล (personalized learning) และการพัฒนาทักษะที่หลากหลาย

งานของ {Labadze, 2023 #13} ยังระบุว่าแชทบอทในบริบทการศึกษาสามารถเพิ่มผลสัมฤทธิ์ทางการเรียน การให้เหตุผลเชิงชัดเจน และการจดจำความรู้ ประโยชน์เชิงการบริหารจัดการก็เกิดขึ้นด้วย ทั้งการให้ความช่วยเหลือทันที การเข้าถึงข้อมูลอย่างรวดเร็ว และการปรับปรุงประสบการณ์ของผู้เรียน ในขณะเดียวกัน การทบทวนวรรณกรรมยังชี้ให้เห็นข้อท้าทายสำคัญ ได้แก่ การที่แชทบอทจำนวนมากถูกนำไปใช้โดยไม่มีกรอบทฤษฎีที่มั่นคง และความกังวลเรื่องจริยธรรมในการใช้งาน AI ในระดับการศึกษา

งานวิจัยอื่นในทิศทางเดียวกันครอบคลุมการนำแชทบอทไปใช้ในโดเมนเฉพาะ เช่น การสอนวิทยาการคอมพิวเตอร์ การเรียนภาษา และการศึกษาทั่วไป ในกรณีของระบบ VIRIYA โดเมนที่เฉพาะเจาะจงคือการให้บริการข้อมูลหลักสูตร (advisory service) ซึ่งมีความแตกต่างจากการใช้เพื่อการเรียนการสอน แชทบอทในบทบาทที่ปรึกษาต้องให้ความสำคัญกับความถูกต้องของข้อมูลเชิงระเบียบและความเป็นปัจจุบันของเอกสารมากกว่าการสร้างประสบการณ์การเรียนรู้ที่ปรับเฉพาะบุคคล ดังนั้นการออกแบบจึงเน้นที่คุณภาพของการสืบค้นและการอ้างอิงแหล่งที่มามากกว่าด้าน personalization

## 2.7 กรอบแนวคิดของการศึกษา

การศึกษานี้บูรณาการทฤษฎีและเทคนิคจากหัวข้อ 2.1 ถึง 2.6 มาออกแบบและประเมินระบบ VIRIYA ตามกรอบแนวคิดที่แสดงในภาพที่ 2.1

[FIGURE 2.1: กรอบแนวคิดของการศึกษา แสดงความสัมพันธ์ระหว่าง 3 ส่วนหลัก (1) Input: เอกสารหลักสูตร + คำถามนิสิต (2) ระบบ VIRIYA ที่ขับเคลื่อนด้วย Agentic RAG + Multi-tenant SaaS (3) Output: คำตอบที่นำไปใช้ได้จริง + การยอมรับตามกรอบ UTAUT + ความเป็นไปได้เชิงพาณิชย์]

องค์ประกอบของกรอบแนวคิดแบ่งเป็นสามส่วน ส่วนแรกคือปัจจัยนำเข้า (Input) ที่ประกอบด้วยเอกสารหลักสูตรของ tenant และคำถามของนิสิต ส่วนที่สองคือระบบ VIRIYA ที่ประมวลผลผ่าน pipeline ของ Agentic RAG ที่ใช้เทคนิคตามงานของ {Lewis, 2020 #1} และ {Gao, 2024 #8} ร่วมกับสถาปัตยกรรม Multi-tenant SaaS ตาม {Bezemer, 2010 #3} ส่วนที่สามคือผลลัพธ์ (Output) สามมิติที่สะท้อนตัวแปรที่การศึกษาวัด ได้แก่ คุณภาพคำตอบที่สามารถนำไปใช้งานจริงจากมุมมองของเจ้าหน้าที่หลักสูตร การยอมรับเทคโนโลยีของนิสิตตามกรอบ UTAUT ของ {Venkatesh, 2003 #14} และความเป็นไปได้เชิงพาณิชย์ที่ขับเคลื่อนด้วยข้อมูล willingness to pay และ buying center

ในเชิงเทคนิคของการวัดการยอมรับเทคโนโลยี ผู้วิจัยเลือกใช้กรอบ UTAUT ของ {Venkatesh, 2003 #14} ครอบคลุมทั้งห้าโครงสร้าง (PE, EE, SI, FC, BI) แทนกรอบ TAM พื้นฐาน เพื่อให้สามารถจับประเด็นด้านอิทธิพลทางสังคมและเงื่อนไขสนับสนุนที่มีนัยสำคัญในบริบทของสถาบันการศึกษา

ความสัมพันธ์ระหว่างสามส่วนเป็นแบบเชิงเส้นตามวงจร Input → Process → Output แต่ภายในระบบ VIRIYA เองมี feedback loop ที่เจ้าหน้าที่สามารถปรับปรุงฐานความรู้ผ่าน Admin Portal เมื่อพบคำถามที่ระบบตอบไม่ได้ การออกแบบแบบ feedback นี้สอดคล้องกับแนวทางของ Modular RAG ที่ {Gao, 2024 #8} ระบุว่าเป็นทิศทางของระบบรุ่นใหม่ที่สามารถปรับปรุงตนเองได้ตามการใช้งาน

กรอบแนวคิดนี้นำไปสู่การออกแบบวิธีวิจัยในบทที่ 3 ที่แบ่งเป็นสามขั้นตอน และเป็นแนวทางในการตีความผลการศึกษาในบทที่ 4 ถึง 6 โดยตัววัดในแต่ละบทถูกเลือกให้สอดคล้องกับส่วนประกอบของกรอบแนวคิดนี้
