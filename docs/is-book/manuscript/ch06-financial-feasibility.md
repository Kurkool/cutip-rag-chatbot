# บทที่ 6 การศึกษาความเป็นไปได้ทางการเงิน

บทนี้วิเคราะห์ความเป็นไปได้ทางการเงินของระบบ VIRIYA ในการดำเนินธุรกิจในรูปแบบบริษัทจำกัดที่ให้บริการ B2B SaaS สำหรับสถาบันการศึกษา ครอบคลุมสินทรัพย์หลัก (6.1) สมมติฐานการเงิน (6.2) โครงสร้างต้นทุน (6.3) ประมาณการรายได้ (6.4) ค่าใช้จ่ายในการขายและการบริหาร (6.5) การลงทุนเริ่มต้นและโครงสร้างเงินทุน (6.6) งบแสดงฐานะทางการเงิน (6.7) งบกำไรขาดทุน (6.8) งบกระแสเงินสด (6.9) การวิเคราะห์จุดคุ้มทุน (6.10) การวิเคราะห์ความอ่อนไหว (6.11) บทสรุปทางการเงินพร้อมตัวชี้วัด NPV/IRR/MIRR/Payback (6.12) และข้อจำกัดของการวิเคราะห์ (6.13) การวิเคราะห์ขยายครอบคลุม 5 ปี (2569-2573) ตัวเลขในบทนี้ประมาณการจากราคาสาธารณะของผู้ให้บริการและการจำลองการใช้งาน อัตราแลกเปลี่ยนที่ใช้ในการคำนวณคือ 1 ดอลลาร์สหรัฐ = 35 บาท

## 6.2 สมมติฐานการเงิน

การศึกษาความเป็นไปได้ทางการเงินของ VIRIYA ตั้งอยู่บนสมมติฐานการดำเนินธุรกิจในรูปแบบบริษัทจำกัดที่จดทะเบียนในประเทศไทย สามารถออกใบกำกับภาษีและสัญญาที่รับการจัดซื้อจัดจ้างจากมหาวิทยาลัยรัฐได้ ข้อสมมติหลักมีดังนี้

1. **รูปแบบและโครงสร้างเงินทุน** บริษัทเริ่มต้นด้วยเงินทุนจดทะเบียน 2,000,000 บาท จากผู้ก่อตั้ง 4 คน ที่ออกหุ้นสามัญจำนวน 2,000 หุ้นมูลค่าหุ้นละ 1,000 บาท
2. **เงินกู้ระยะยาว** บริษัทกู้เงินจากหน่วยงานภาครัฐเพื่อส่งเสริม startup เทคโนโลยีจำนวน 1,000,000 บาท อัตราดอกเบี้ย 5 เปอร์เซ็นต์ต่อปี ระยะเวลาคืนเงินกู้ 5 ปี โดยชำระแบบงวดเท่ากันรายปี
3. **รวมเงินลงทุนเริ่มต้น** 3,000,000 บาท (เงินทุนผู้ถือหุ้น 2M + เงินกู้ระยะยาว 1M)
4. **รูปแบบสำนักงาน** บริษัทเช่าพื้นที่ในรูปแบบ Co-working Space เพื่อลด Capital Expenditures และใช้โมเดล Operating Expenses แทน
5. **นโยบายปันผล** จ่ายปันผล 30 เปอร์เซ็นต์ของกำไรสุทธิเริ่มตั้งแต่ปีที่ 4 เพื่อให้มีกำไรสะสมเพียงพอสำหรับการขยายธุรกิจในช่วงแรก
6. **ระยะเวลาการวิเคราะห์** 5 ปี (2569-2573) เพื่อให้สามารถวิเคราะห์ NPV, IRR, MIRR และ Payback Period ได้อย่างเหมาะสมกับธุรกิจ SaaS

สมมติฐานเชิงปริมาณสำหรับการพยากรณ์ 5 ปีแสดงในตารางที่ 6.3

## 6.3 โครงสร้างต้นทุน (Cost Structure)

ต้นทุนของระบบ VIRIYA แบ่งออกเป็นสามประเภทตามลักษณะการเปลี่ยนแปลงเมื่อมีการใช้งาน ได้แก่ ต้นทุนผันแปรต่อ tenant ต้นทุนคงที่ของระบบ และต้นทุนบุคลากรในการดูแลระบบ

| Claude Opus 4.7 input token | $5/M tokens | ~6M tokens (3K คำถาม × 2K context) | ~฿1,050 |

### 6.3.2 ต้นทุนคงที่ของระบบ

ต้นทุนคงที่เป็นส่วนที่ไม่เปลี่ยนแปลงตามจำนวน tenant ที่ให้บริการ (หรือเปลี่ยนแปลงเล็กน้อย) รวม 3,500 บาทต่อเดือน (Pinecone Standard plan 1,750 + GCP Cloud Run 1,050 + Firestore+Storage 350 + Domain+Monitoring 350) ต้นทุนคงที่นี้กระจายไปยัง tenant ที่ใช้งาน เป็นลักษณะของ economies of scale ที่ทำให้สถาปัตยกรรม Multi-tenant มีความคุ้มค่าเพิ่มขึ้นเมื่อจำนวนลูกค้าเพิ่มขึ้น

## 6.10 การวิเคราะห์จุดคุ้มทุน (Break-even Analysis)

## 6.13 ข้อจำกัดของการวิเคราะห์การเงิน

การวิเคราะห์ในบทนี้มีข้อจำกัดที่สำคัญหลายประการที่ผู้อ่านควรตระหนักก่อนการตัดสินใจเชิงการเงินจริง

ข้อจำกัดแรกคือตัวเลขต้นทุน API คำนวณจากราคาสาธารณะของผู้ให้บริการและสมมติฐานการใช้งานเฉลี่ย 3,000 คำถามต่อ tenant ต่อเดือน ไม่ใช่ข้อมูลการใช้งานจริงจากการ deploy ที่เพิ่งเริ่มในเดือนเมษายน 2569 เมื่อระบบดำเนินการครบ 12 เดือน ผู้ประกอบการสามารถ refine ตัวเลขเหล่านี้จาก Anthropic console, Cohere console, Pinecone console, และ GCP billing dashboard

ข้อจำกัดที่สองคือสมมติฐานการกระจายลูกค้าตาม tier (Starter 60% / Pro 30% / Enterprise 10%) เป็นการประมาณการจากลักษณะของตลาดเป้าหมายที่ผู้วิจัยเข้าใจ แต่ยังไม่ได้ผ่านการทดสอบกับลูกค้าจริงจำนวนมาก การกระจายจริงอาจเบี่ยงเบนไปทาง Starter หากตลาดมีความอ่อนไหวต่อราคาสูงกว่าที่คาด ซึ่งจะลด ARPU และกระทบ NPV

ข้อจำกัดที่สามคือการวิเคราะห์ไม่ได้รวมต้นทุนการได้มาของลูกค้า (Customer Acquisition Cost หรือ CAC) ที่จะเกิดขึ้นเมื่อธุรกิจขยายสู่ลูกค้านอกเครือข่ายเดิม สำหรับธุรกิจ SaaS ที่ขายแบบ B2B การลงทุนด้านการตลาดและการขายมักอยู่ในช่วง 20-40 เปอร์เซ็นต์ของรายได้ในช่วงแรก ซึ่งอาจเพิ่ม SG&A สูงกว่าที่ประมาณการไว้

ข้อจำกัดที่สี่คือการวิเคราะห์ไม่ได้พิจารณาความเสี่ยงจากการเปลี่ยนแปลงราคา API ของผู้ให้บริการ โดยเฉพาะ Anthropic ที่มีแนวโน้มการปรับลดราคาต่อเนื่องในช่วงปี 2567-2568 (Opus 4.5 ลดราคา 66 เปอร์เซ็นต์จาก Opus 4) และความเสี่ยงจากอัตราแลกเปลี่ยนที่การวิเคราะห์สมมติคงที่ที่ 35 บาทต่อดอลลาร์สหรัฐ ธุรกิจที่ดำเนินงานในไทยแต่ใช้บริการ API ที่คิดราคาเป็นดอลลาร์ต้องบริหารความเสี่ยงอัตราแลกเปลี่ยนในระยะยาว

ข้อจำกัดที่ห้าคืองบแสดงฐานะทางการเงิน งบกำไรขาดทุน และงบกระแสเงินสดเป็นการประมาณการที่ยังไม่ได้ผ่านการตรวจทานโดยนักบัญชี ตัวเลขในส่วนของลูกหนี้การค้า เจ้าหนี้การค้า และค่าใช้จ่ายค้างจ่ายเป็นการประมาณตามสัดส่วนของรายได้และค่าใช้จ่าย ซึ่งในการดำเนินธุรกิจจริงจะขึ้นอยู่กับนโยบายเงื่อนไขการชำระเงินและระบบการเรียกเก็บ

ข้อจำกัดสุดท้ายคือการคำนวณ NPV ใช้ WACC แบบคงที่ที่ 12 เปอร์เซ็นต์ตลอด 5 ปี ซึ่งในความเป็นจริง cost of capital อาจเปลี่ยนแปลงตามสภาวะดอกเบี้ยในตลาดและความเสี่ยงของบริษัทที่เปลี่ยนไปตามวุฒิภาวะทางธุรกิจ

ตัวเลขและสมมติฐานทั้งหมดในบทนี้ถูกออกแบบให้เป็นจุดเริ่มต้นของการวางแผนทางการเงินมากกว่าการพยากรณ์ที่แม่นยำ ผู้วิจัยแนะนำให้ผู้ประกอบการที่สนใจนำผลการศึกษาไปใช้ดำเนินการทดสอบสมมติฐานเหล่านี้ผ่านการสัมภาษณ์ลูกค้าเป้าหมายและการจำลองโมเดลการเงินแบบละเอียดอีกครั้งก่อนการตัดสินใจลงทุน
